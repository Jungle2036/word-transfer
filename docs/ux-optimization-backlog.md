# UX 优化待办

> 记录于 2026-07-06，暂不实现。
> **目标平台**: Windows .exe（通过 pkg 构建），典型场景是双击运行或拖拽文件到 exe。

---

## 🔴 高优先级（Windows .exe 关键问题）

### 1. 报错时控制台窗口直接消失
- **现状**: `maybePauseBeforeExit` 仅在 `noArgs && win32` 时暂停。如果用户在命令行传参运行（或快捷方式带参），出错后窗口立即关闭，看不到任何错误信息。连 `--help` 都看不到就闪退了。
- **方案**: Windows 下**永远**在退出前暂停（无论有无传参），或至少在任何 `process.exit(1)` 之前都暂停。
- **影响文件**: `word.js` maybePauseBeforeExit, main catch 块

### 2. 不支持拖拽文件到 exe
- **现状**: 拖拽 Excel 到 exe 上时，Windows 把文件路径作为**无前缀**的位置参数传入。当前 CLI 只解析 `--key=value` 和 `--key value` 格式，位置参数被忽略。
- **方案**: 支持位置参数自动推断：第一个 `.xlsx/.xls` 当 `--excel`，第一个 `.docx` 当 `--template`。
- **影响文件**: `word.js` parseCliArgs

### 3. 输出文件与 exe 同目录
- **现状**: 输出文件生成在 `process.cwd()`（exe 所在目录）。Windows 下 exe 可能在 Program Files 或桌面，生成一堆文件会污染目录，且 Program Files 可能有写入权限问题。
- **方案**: 默认输出到**桌面**或 exe 同级的 `output/` 目录，或生成完成后打开所在文件夹。
- **影响文件**: `word.js` main

### 4. Windows 控制台中文乱码
- **现状**: 中文 Windows 控制台默认编码为 GBK（代码页 936），Node.js 输出的 UTF-8 中文可能显示乱码。
- **方案**: 启动时执行 `chcp 65001` 切换到 UTF-8，或用 `iconv-lite` 做编码转换。
- **影响文件**: `word.js` main 入口

### 5. 缺少 `--help`
- **现状**: 双击 exe 的新用户完全不知道要准备什么文件、放哪里。
- **方案**: 加 `--help` / `-h`，输出中文参数说明 + 使用示例。Windows 下还应包含「如何准备文件」的引导。
- **影响文件**: `word.js` parseCliArgs / main

## 🟡 中优先级（交互体验）

### 6. 交互模式引导不够友好
- **现状**: 只问「请输入模板路径」，新用户不知道模板是什么、长什么样、放哪里。
- **方案**: 首次运行时检测模板文件是否存在，不存在则给出友好提示：「请将 template.docx 和 template_excel.xlsx 放在与本程序相同的目录下」。

### 7. 大数据量无进度提示
- **现状**: 处理大量数据时控制台一动不动，用户可能以为卡死了。
- **方案**: 输出 `正在处理... X/Y 行` 进度，或至少「正在生成，请稍候...」

### 8. 报告头信息硬编码
- **现状**: 制造单位、批次号、检验人、检定日期等无法修改。
- **方案**: 从输入 Excel 的特定列读取，或交互模式下逐项询问。

## 🟢 低优先级（锦上添花）

### 9. 同名文件覆盖风险
- **现状**: 时间戳精确到秒，同一秒跑两次会覆盖。
- **方案**: 毫秒级时间戳或检测重名自动加 `(1)` 后缀。

### 10. 生成完成后打开文件夹
- **现状**: 用户要手动找输出文件。
- **方案**: Windows 下 `start "" "输出目录"` 自动打开文件夹。

### 11. 随机种子可选
- **现状**: 每次结果不同，质检复核无法复现。
- **方案**: `--seed` 参数固定随机序列。

### 12. 缺少图标和版本信息
- **现状**: exe 没有图标，右键属性看不到版本号。
- **方案**: pkg 构建时指定 `--icon` 和 `--version`。
